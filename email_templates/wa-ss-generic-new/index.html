Here are a few
recent developments

1) Quick replies - Header is now optional

To bring in consistency with the WhatsApp implementation, we have now made header optional in quick replies, it was previously mandatory. There is a change in curl for interactive session media. You can read about it here.

2) Embed registration - failure journey

User journey if in case setup fails for whatever reason after completing the FB journey, they were not aware of the failure. A loader is now added to the user journey before taking them to the Congratulations screen and only if its success, user can continue the embed journey. If failure, user will need to contact devsupport@gupshup.io.

3) Embed registration - emails

User will now receive emails at every milestone completed by user in GS onboarding go live journey (except their Facebook interface journey)

4) Audio files support in Customer Support tool

We have now added audio file support for agents to send audio files to users in Customer support tool. File types supported: audio/aac, audio/mp4, audio/ogg, audio/mpeg, codecs=opus (The base audio/ogg type is not supported by FB).

5) Cyrillic character support in Billing

For our customers speaking languages which require Cyrillic character support in their script, are now supported on Gupshup wallet billing.

6) Apps can be linked only to approved partners, and never to rejected partners

Earlier apps could be linked to any partner ID, but now only approved partner IDs are accepted to link the app to it and rejected partners are directly unlinked if they are linked.

Bug fixes
  • Fixes for media with interactive messages.
  • Fixes for partner ID getting changed automatically for certain apps.
  • Bug fixes related to OTP verification

Announcements / Notices

Major WhatsApp outage on Oct 04, 2021

As you might be aware, on October 4, 2021, WhatsApp experienced a disruption to all their apps and products. Access to their services, including the WhatsApp Business API, Ads that Click to WhatsApp, and the WhatsApp Business App, have since been fully restored as confirmed by them. WhatsApp business messages (from both the WhatsApp Business API and WhatsApp Business App) did not deliver during the outage but were delivered since the outage was resolved. All messages that were scheduled to be sent during the outage have since been delivered, as per WhatsApp.

WhatsApp's recent email to partners said - "We know how challenging the inability to communicate with customers can be, and we are working to address what caused this system-wide outage to prevent this from happening again. We sincerely apologize for the inconvenience this caused and appreciate your understanding and patience during this time."

Once this outage was completed, Gupshup ensured that all enqueued messages and processes started flowing smoothly. If any of you faced issues related to this, please escalate immediately.
