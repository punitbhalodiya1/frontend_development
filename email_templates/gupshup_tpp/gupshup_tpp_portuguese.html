Você está ciente do Programa para Provedores de Tecnologia (TPP, em inglês), de como ele está conectado à CAPI (API de nuvem), e como a Gupshup o está tornando mais fácil para você?

Há muito entusiasmo sobre o TPP entre os ISVs. Embora tenha sido lançado pela Meta em outubro de 2023, ISVs já associados com a Gupshup terão tempo para se tornar Provedores de Tecnologia até dezembro de 2024. Só a partir de 29 de janeiro de 2024 é que será obrigatório que novos ISVs tornem-se Provedores de Tecnologia antes de começar a integrar contas WABA com a Gupshup.

A Meta também começou a transição para se tornar uma API inteiramente na nuvem (CAPI), hospedada pela plataforma WhatsApp Business. A descontinuação das contas WABA com hospedagem local está planejada para acontecer até o quarto trimestre do ano que vem.

O TPP e a API de nuvem hospedada pela Meta estão diretamente ligados. Com o TPP, novos parceiros somente poderão integrar novas contas WABA na API de nuvem. Para parceiros existentes, ainda será possível integrar contas WABA com hospedagem local até sua incorporação no programa TPP ou, no mais tardar, até 30 de junho de 2024.

Abaixo, você encontra uma explicação transparente e detalhada sobre tudo isso.

Tudo sobre o TPP

A Meta está abrindo seu ecossistema para desenvolvedores terceirizados (ISVs), que geram soluções com valor agregado sobre a API do WhatsApp.

As empresas passarão a poder incorporar a plataforma WhatsApp Business através dos Parceiros de Soluções existentes da Meta (BSPs) e/ou através de provedores de tecnologia.

Opções de cobrança para TPP:

Opção 1: Diretamente com a Meta

Pontos-chave a notar nesta opção:

O cliente paga diretamente para a Meta

O ISV precisa apenas hospedar o cadastro incorporado em seu site

O TPP apenas suporta a hospedagem na API da nuvem, da Meta

O ISV gerencia toda a mensageria, incluindo filas, perda de mensagens, resolução prioritária com a Meta e suporte para campanhas grandes, entre outros.

NÃO INCLUÍDOS: Serviços de valor agregado da Gupshup, como filas de mensagens, gerenciamento de mensagens, atendimento 24h, soluções efetivas, relatórios personalizados, etc.
Opção 2:
Compartilhamento de conta com a Gupshup:
o ISV hospeda o SDK de integração

Pontos-chave a notar nesta opção:

Os clientes não pagam diretamente para a Meta. Os parceiros terão a opção de deixar ou o cliente ou o próprio parceiro pagar para a Gupshup.

O TPP suporta apenas a hospedagem na API de nuvem, da Meta

O ISV precisará hospedar o cadastro incorporado em seu próprio portal

O ISV terá a possibilidade de deixar a Gupshup se encarregar da gestão de mensagens, podendo concentrar-se nos serviços centrais de seu negócio

Ao trabalhar com a Gupshup, o ISV também obterá acesso ao stack total de comunicações e à plataforma de engajamento com o cliente da Gupshup, que está constantemente sendo aperfeiçoada para a gestão do ciclo de vida de comunicações dos parceiros

INCLUI: Os serviços com valor agregado da Gupshup, como filas de mensagens, gestão de mensagens, atendimento 24h, soluções efetivas, relatórios personalizados, etc.

INCLUI: Suporte direto da Gupshup e também da Meta, dado que a Gupshup, como parceira de alta valia, conta com o mais alto nível de suporte dentro da Meta

INCLUI: Acesso aos serviços de valor agregado da Gupshup, como portal de parceiros com analytics para comunicações e contas WABA relevantes

Tudo sobre a API de nuvem

Vamos repassar o anúncio recente da Meta -

A partir de 9 de janeiro de 2024, todas as novas atualizações de funcionalidades serão direcionadas exclusivamente para a API de nuvem. A versão com hospedagem local continuará a receber lançamentos de versões trimestrais, mas apenas para corrigir bugs e erros de segurança, dado que o esforço para desenvolvimento de produtos está investido na API de nuvem

A partir de 1° de julho de 2024, não será mais possível registrar novos números de telefone através da hospedagem local. Todos os novos números de telefone após esta data serão suportados exclusivamente na API de nuvem. Números existentes com hospedagem local poderão permanecer nela até a data de descontinuação da hospedagem local.

Em 23 de outubro de 2025, a versão local final (v2.63) expirará. Mensagens enviadas de ou para negócios através da API local não serão mais entregues, após esta data.

Os parceiros poderão continuar incorporando novas contas WABA em hospedagem local, assim como na API de nuvem, até serem integrados no TPP ou, no mais tardar, até 30 de junho de 2024. A Gupshup tomará medidas para realizar incorporações de novas contas na API de nuvem nos próximos meses, através da interface do usuário. Para contas WABA existentes em hospedagem local, a Gupshup também vai planejar a migração para a API de nuvem mais tarde neste ano — você será informado.

Gupshup em prontidão

Gostaríamos de aproveitar a oportunidade para garantir que você pode ficar tranquilo em relação ao Programa para Provedores de Tecnologia, a API de nuvem e todas as mudanças requeridas nos seus sistemas, já que o prazo para você será somente dezembro de 2024.

Embora o prazo ainda esteja longe, a Gupshup já planejou uma transição suave para você ao longo dos próximos trimestres.

Em breve, a Gupshup introduzirá um portal de parceiros unificado para este fim. Dentro de algumas fases, seu portal de parceiros atual passará por um upgrade com as seguintes capacidades:
1

Nova incorporação de contas WABA no portal de parceiros (não será preciso usar o portal self-service para o onboarding)
2

As contas de parceiros gerenciadas pela Gupshup poderão administrar suas carteiras diretamente no portal de parceiros (também aqui, não será mais preciso passar pela carteira self-service).
3

Os parceiros da Gupshup que gerenciam suas próprias contas Gupshup também poderão ser integrados através do próprio portal dos parceiros, e poderão ser convidados para um portal especificamente para o onboarding de contas WABA, com a carteira e a carteira do próprio cliente.
4

Os parceiros atuais também poderão fazer o onboarding de contas WABA com configuração como TPP (ID de solução) através do mesmo portal.

É bastante informação a processar, mas garantimos que também estaremos informando você melhor através de webinars e workshops para explicar tudo em mais detalhes.

Para mais informações, pedimos que fale com o representante da sua conta na Gupshup. Nosso compromisso é garantir que sua migração transcorra sem inconvenientes ou imprevistos.
