Importante cambio en el flujo de activación

OBO Sunset

Al activar un nuevo número de teléfono o migrar uno desde otro BSP, los partners ya no dispondrán de la opción OBO (en nombre de) de onboarding. A partir de ahora, el registro integrado será el único método de incorporación. Para las aplicaciones OBO existentes, pronto proporcionaremos un método para migrar al registro integrado.

On-premise sunset

Al lanzar un nuevo número de teléfono o migrar uno desde otro BSP, los partners ya no dispondrán de la opción de alojamiento Onpremise. A partir de ahora, la API Cloud será la única opción de alojamiento. Para las apps on-premise existentes, ponte en contacto con el Servicio de Atención al Partner para migrarlas a la API Cloud.

Nota: Asegúrate de configurar aquí los endpoints de envío de mensajes V2 para facilitar la transición a la API Cloud. El error relacionado con el endpoint V2 (donde los mensajes con emojis en el menú de la lista fallaban) también se solucionará con esta versión.

Cambios en la migración desde otro BSP a Gupshup

Junto con los cambios anteriores, el onboarding de un nuevo número de teléfono o tras la migración desde otro BSP - para ambos casos, el usuario DEBE seleccionar el caso de uso correcto al inicio del flujo para garantizar que se toman las medidas de backend correctas para cada caso. Esto garantizará que las plantillas y otros detalles se obtengan completamente del FBM en caso de migración y que el servicio de asistencia pueda proporcionar una ayuda más rápida y pertinente.

Nota: Para migrar un número de teléfono OBO del BSP de origen a Gupshup, te pedimos que realices la migración de OBO a Registro Integrado en el BSP de origen, antes de iniciar la migración en Gupshup.

Entrega o Lectura basada en la deducción de tarifas de WhatsApp

Gupshup empezará a cobrar la tarifa de WhatsApp en el estado de entrega o de lectura, lo que ocurra primero. Nota: la tarifa de Gupshup se seguirá deduciendo cuando se envíe el mensaje.

Límites de sobregiro preaprobados

También estamos introduciendo límites de sobregiro reclamables preaprobados basados en el uso y el historial de recargas del monedero.

Gupshup asignará ahora un límite de sobregiro precalculado y aprobado cada principio de mes, que se mostrará en la interfaz del monedero. Este límite se basa en los patrones de uso y recarga de los últimos 3 meses. Un usuario puede reclamar un límite igual o inferior al límite calculado, que se mostrará como límite "asignado".

Nota: Si Gupshup ha asignado un límite manualmente, éste se aplicará independientemente del límite calculado.

Consulta aquí la guía para saber cómo funcionan los límites de sobregiro preaprobados

La API Cloud ya está disponible en Turquía

Meta ha anunciado que la API Cloud ya está disponible en Turquía. Esto significa que las empresas de Turquía pueden empezar a utilizar WhatsApp, y los usuarios con números turcos ya pueden enviar y recibir mensajes a través de la API Cloud. Esto significa que los partners que prestan servicios a empresas turcas también pueden formar parte del Tech Provider Program.

Proceso de configuración de Tech Provider para partners no pertenecientes al TPP

El Portal de Partners permitirá a los partners que no hayan completado su onboarding al Tech Provider Program en Gupshup, completar su configuración y mapear su nuevo número de teléfono o migración de registro con su Joint Solution ID para cualquier app vinculada a partners. Lee esta guía para saber más.

Creación de app de demostración para Tech Providers

Los Tech Provider de Gupshup ahora pueden marcar una app como demo (para uso propio del partner) en el Partner Portal. En el momento de la puesta en marcha, podrán realizar el onboarding en su propio FBM sin registrar la solución conjunta. Lee esta guía para saber más.

Actualizaciones de Meta

Restricción de conversaciones de marketing por usuario

A partir de este mes, Meta introdujo nuevas medidas para mejorar la interacción de las plantillas de marketing a nivel global en WhatsApp, limitando el número de mensajes que una persona recibe de cualquier empresa en un período determinado. Consulta esta guía para saber más.

Categorización de plantillas:

A partir de junio de 2024, Meta identificará las plantillas que considere que no se ajustan a la categoría correcta y notificará a los propietarios de la WABA con un mes de antelación sobre el cambio identificado que los usuarios deben esperar en el próximo mes. Este proceso tendrá lugar el día 1 de cada mes. Consulta nuestro correo electrónico ' Actualización de Meta: Cambios Mensuales en las Categorías de Plantillas' para informarte al respecto.
