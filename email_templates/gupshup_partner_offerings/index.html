New Developments on Gupshup

Facebook Cloud API (FBC) option now available for new WABA onboarding 

Now you get the option to select the hosting option as FBC (CAPI / Cloud API hosted by Meta) while onboarding a new WABA in the Self-Serve portal. The existing On-prem options - India and Germany will also continue to show if you select ‘Gupshup’ hosting.

Note: Meta will only allow onboarding WABAs on Cloud API (and not on-prem) starting June 2024.

Seamless Migration

Are you looking to migrate your current WhatsApp Business Account (phone number) from another BSP to Gupshup? You'll be happy to know that this is now a seamless process through Embedded Signup. Click here to learn more about how to make the switch.

Meta Updates
1

Frequency Capping for India from Feb 06

META launched a new integrity check on 06 Feb 2024, called 'Frequency Capping'. This check ensures that an Indian end user (with a +91 phone number) can only receive messages from 'X' brands in a rolling Y-day window. (The x and y are currently being dynamically decided by META and are not to be known to businesses or BSPs). This feature might roll out globally once the tests are completed.
2

Read rate consideration from April 01

META is looking to empower businesses to send more engaging and relevant marketing messages. Thus they will start considering read rates alongside traditional metrics like blocks and reports. Campaigns with low read rates may be temporarily paused, starting April 1, 2024, allowing businesses to refine templates before scaling volume. This will provide businesses time to iterate on the templates with the lowest engagement before scaling volume.
3

New Conversational Features on Cloud API

META also introduced new conversational components on Cloud API globally, making automated experiences more intuitive. These features include welcome messages, icebreakers, and commands, enhancing in-thread experiences for consumers. Currently, this is possible through WhatsApp Manager.
