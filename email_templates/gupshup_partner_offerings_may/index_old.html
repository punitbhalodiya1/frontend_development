Recent Developments on Gupshup

Upcoming changes next week

Compare Templates via UI on WhatsApp Self-Serve

Now you can compare two different templates to find out which one has lower block rates.

Text formatting support in template creation

Text formatting will be supported in template creation. This enhancement allows you to make your template text more organized and readable, highlighting any important information effectively.

Major go live flow changes

OBO sunset

While taking a new phone number live or migrating one from another BSP, partners will no longer get the OBO (on behalf of) option of onboarding. Embedded signup will be the only method of onboarding going forward. For existing OBO apps, we will soon provide a method to migrate to Embedded Signup.

Onpremise sunset

While taking a new phone number live or migrating one from another BSP, partners will no longer get the hosting option as On-premise. Cloud API will be the only hosting option going forward. For existing on-premise apps, please reach out to Partner Support for migrating them to Cloud API.

Note: Please ensure to set up the V2 send message endpoints here, for smooth transitioning to Cloud API. Bug related to V2 endpoint (where messages with emojis in list menu were failing) will also be fixed with this release.

Changes in migration from other BSP to Gupshup

Along with the above changes, the new phone number onboarding or after migration from other BSP - for both these cases, user MUST select the correct use-case at the start of the flow to ensure the right backend steps are taken for each case. This will ensure templates and other details are fetched completely from FBM in case of migration and support is able to provide quicker and relevant assistance.

Note: For migrating an OBO phone number from source BSP to Gupshup, we request you to get the OBO to Embedded Signup migration done at source BSP, before initiating migration at Gupshup.

Check out the guide here to know how the Go Live flow will work

Delivery or Read based WhatsApp fee deduction

Gupshup will start charging the WhatsApp fee on either delivery or read status, whichever occurs first. Note: the Gupshup fee will still be deducted when the message is sent.

Pre-approved overdraft limits

We are also introducing pre-approved claimable overdraft limits based on usage and recharge history of the wallet.

Gupshup will now assign a pre-calculated and approved overdraft limit every start of the month, which will be displayed in the wallet interface. This limit is based on the last 3 months of usage and recharge patterns. A user can claim the limit equal to or less than the calculated limit which will show as the ‘assigned’ limit.

Note: If Gupshup has assigned a limit manually, it will be enforced regardless of the calculated limit.

Check out the guide here to know how the the preapproved overdraft limits will work

Cloud API now live in Türkiye

Meta announced that the Cloud API is now live in Türkiye. This means businesses in Türkiye can get started on WhatsApp, and users with Turkish numbers can now send and receive messages via the Cloud API. This means that partners serving Turkey businesses can now be part of the Tech Provider Program as well.

Tech Provider setup process for non-TPP partners

Partner Portal will allow partners who have not completed their Tech Provider Program onboarding on Gupshup, to now complete their setup and map their new phone number or migration onboarding with their Joint Solution ID for any partner linked app. Read this guide to know more

Demo app creation for Tech Providers

Tech Provider on Gupshup can now mark an app as a demo app (for partner’s own use) on Partner Portal. On go live, they will be able to do the onboarding on their own FBM without registering the joint solution. Read this guide to know more

Meta Updates

Per-User marketing conversation restriction

Starting this month, Meta introduced new measures to improve marketing template engagement globally on WhatsApp by limiting the number of messages a person receives from any business within a given period. Refer this guide to know more

Template Categorization:

Starting June 2024, Meta will be identifying templates that they believe do not fit the right category based on template category guidelines and will be notifying WABA owners with a month's notice about the identified change that users must expect in the coming month. This process will occur on the 1st of each month. Refer our emailer ‘Meta Update: Monthly Changes to Template Categories’ to know about the same.
