Principais mudanças no fluxo de ativação

Descontinuação do fluxo OBO (em nome de)

Ao ativar um novo número de telefone ou migrar um número de outro BSP, os parceiros não contarão mais com a opção de onboarding OBO (em nome de). O cadastro incorporado será a única forma de onboarding de agora em diante. Para apps existentes no modo OBO, em breve providenciaremos um método para migrar para o cadastro incorporado.

Descontinuação da hospedagem local

Ao ativar um novo número de telefone ou migrar um número de outro BSP, os parceiros não contarão mais com a opção de hospedagem local. A API de nuvem será a única opção de hospedagem de agora em diante. Para apps que no momento funcionam com hospedagem local, pedimos que entre em contato com nosso atendimento para parceiros a fim de migrá-los para a API de nuvem.

Obs.: Certifique-se de configurar os endpoints V2 para envio de mensagens aqui, a fim de garantir uma transição fácil para a API de nuvem. Com este lançamento, também será corrigido o bug conectado com o endpoint V2 (onde mensagens com emojis no menu em lista não estavam sendo enviadas).

Mudanças na migração a partir de outro BSP para a Gupshup

Em ambos os casos acima, o onboarding de novos números ou a migração posterior de outro BSP, o usuário DEVE selecionar o caso de uso correto logo no início do fluxo. Isto garante que as etapas corretas sejam tomadas no backend para cada caso, para que os modelos e outros detalhes sejam obtidos completamente do Gerenciador de Negócios do Facebook (no caso de migração). Isto também permitirá que a equipe de suporte ofereça assistência mais rápida e relevante.

Obs.: Para migrar um número de telefone no formato OBO para a Gupshup a partir do BSP original, pedimos que você complete a migração de OBO para cadastro incorporado com o BSP original, antes de iniciar a migração para a Gupshup.

Confira nosso guia aqui para saber como o fluxo de ativação funcionará.

Aplicação de taxa WhatsApp com base em status "Recebido" ou "Visto"

A Gupshup começará a cobrar a taxa do WhatsApp após a mensagem adquirir o status de "recebida" ou "vista" — seja qual for o primeiro. Obs.: A taxa da Gupshup continuará sendo aplicada quando a mensagem for enviada.

Limites de crédito pré-aprovados

Também estamos introduzindo limites pré-aprovados de crédito disponível, os quais se baseia no histórico de uso e recargas da carteira.

A Gupshup passará a atribuir um limite pré-calculado e aprovado no começo de cada mês, o qual será exibido na interface da carteira. O limite será baseado nos 3 meses anteriores de uso e nos hábitos de recarga. O usuário pode usufruir de uma quantia igual a ou inferior ao limite calculado, que será exibido como "Limite atribuído".

Obs.: Se a Gupshup tiver atribuído um limite manualmente, ele será aplicado independentemente do limite calculado.

Confira o guia aqui para saber como vão funcionar os limites de crédito pré-aprovados.

A API de nuvem já está ativa na Turquia

A Meta anunciou que a nova API de nuvem já está ativa na Turquia. Isto quer dizer que as empresas na Turquia podem começar a usar o WhatsApp, e usuários com números turcos já podem enviar e receber mensagens através da API de nuvem. Com isso, os parceiros que trabalham com negócios na Turquia também já podem participar do Programa de Provedores de Tecnologia.

Processo de configuração para provedores de tecnologia que não são parceiros TPP

O Portal de Parceiros permitirá aos parceiros que ainda não completaram seu onboarding no Programa de Provedores de Tecnologia com a Gupshup que completem sua configuração e mapeiem o novo número de telefone ou onboarding de migração com seu ID de solução conjunta para qualquer app conectado a um parceiro. Leia este guia para saber mais.

Criação de aplicativo demo para provedores de tecnologia

Agora, os provedores de tecnologia na Gupshup já podem marcar um app como app para demonstração (para uso do próprio parceiro) no Portal de Parceiros. Após ativar, eles poderão fazer o onboarding em sua própria conta de Gerenciador de Negócios do Facebook, sem registrar a solução conjunta. Leia este guia para saber mais.

Atualizações da Meta

Restrição de conversações de marketing por usuário

Começando este mês, a Meta introduziu novas medidas para melhorar o engajamento das mensagens de marketing a nível global, limitando o número de mensagens que uma pessoa pode receber de qualquer empresa dentro de um determinado período. Leia este guia para saber mais.

Categorização de modelos

A partir de junho de 2024, a Meta identificará modelos que não parecem se enquadrar na categoria certa com base nas diretrizes para categorias de modelos e notificará os proprietários das contas WABA, com um mês de prazo, sobre a alteração que deve ser feita dentro do próximo mês. O processo ocorrerá no dia 1° de cada mês. Confira nosso e-mail "Atualização da Meta: Alterações mensais a categorias de modelos" para saber mais sobre o processo.
